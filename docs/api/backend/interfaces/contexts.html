backend.interfaces.contexts

Unified ctx.data callback context types for the Argus hook system.

Every hook — lifecycle, tick, and signal — receives a single ctx parameter whose concrete type is one of the three dataclasses below. Hooks access their per-hook payload through ctx.data (typed via Generic[T] or an explicit field type). Shared services such as config, database, driver, and engine are optionally available on the same object.

Pattern

def on_before_start(ctx: ScriptContext[SystemMetrics]) -> None:
    print(f"CPU at {ctx.data.cpu.usage_percent}%")
    if ctx.config:
        db.write_snapshot(ctx.data)
